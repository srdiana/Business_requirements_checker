Проверь соответствие требований (requirements) шаблону (template) бизнес-требований. Выдели все несоответствия, ошибки, пробелы, а также дай краткое резюме.

Для каждого найденного несоответствия обязательно заполни поля 'suggestion' (конкретная рекомендация по исправлению) и 'justification' (обоснование, почему это важно).

Ответь СТРОГО только в формате JSON, без пояснений, без текста до и после, без markdown-блоков, без комментариев. Просто выведи JSON-объект.

Структура: {"errors": [...], "summary": "..."}

Пример валидного json:
{
    "errors": [
        {
            "category": "Ошибка в структуре",
            "location": "Страница 1",
            "message": "Неверный формат",
            "suggestion": "Попробуйте добавить содержание",
            "justification": "Нет ни одного раздела"
        }
    ],
    "summary": "БТ некорректно"
}

Если не можешь вернуть валидный JSON, верни {"errors": [], "summary": ""}.
Никогда не обрывай JSON, всегда закрывай все скобки и кавычки.
Не добавляй никаких пояснений, комментариев, markdown или текста до/после JSON.

Шаблон:
{template}

Требования:
{requirements}
